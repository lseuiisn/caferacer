# WayPoint MVP 아키텍처

## 1. 제품 구조

WayPoint는 카페 검색만 제공하는 앱이 아니라 카페 탐색, 검수된 드라이브 코스,
커뮤니티와 크루 활동을 하나의 계정과 주행 기록으로 연결하는 플랫폼이다.

하단 탭은 다음 다섯 개로 고정한다.

| 탭 | 핵심 기능 |
| --- | --- |
| 홈 | TMAP 지도, 현재 위치 주변 카페, 주차·가격·키워드 필터, 즐겨찾기, 카페 상세 |
| 코스 | 전체 사용자 공통 일일 추천 3개, 지역·강변·와인딩 등 탐색, 코스 상세, 랭킹 |
| 커뮤니티 | 사진·글 게시, 댓글, 좋아요, 신고·차단 |
| 크루 | 공개/초대 크루, 가입·운영, 크루 대화, 크루 코스와 랭킹 |
| 프로필 | 닉네임·차종·연결 계정, 주행 기록, 작성 글, 즐겨찾기 |

## 2. 코스 주행 흐름

1. 사용자가 코스를 고르고 `코스 시작`을 누른다.
2. 앱은 정밀 위치와 백그라운드 위치 권한, 위치정보 동의를 확인한다.
3. `POST /drive-records`로 기록을 먼저 만든다.
4. `POST /courses/{id}/navigation`에서 TMAP 실행용 시작점·경유지·도착점을 받는다.
5. Flutter가 외부 TMAP 앱을 실행하고 TMAP이 길안내를 담당한다.
6. WayPoint는 백그라운드에서 GPS를 수집해 묶음 단위로 서버에 전송한다.
7. 완료 시 서버가 CoursePath 통과율과 필수 경유지 통과를 검증한다.
8. 유효한 완주 기록만 코스 또는 크루 랭킹에 반영한다.

`CoursePath`와 `CourseNavigationAnchor`는 목적이 다르다.

- `CoursePath`: 우리 DB가 보유하는 전체 경로. 지도 폴리라인과 완주 검증에 사용한다.
- `CourseNavigationAnchor`: 외부 TMAP에 전달하는 소수의 핵심 지점이다.

## 3. GPS와 랭킹 정책

- 백그라운드 위치 권한은 코스 기록과 랭킹 참여에 필수다.
- GPS 속도는 법적 과속 판정이 아니라 기록 검증과 안전 안내에만 사용한다.
- 정확도 50m를 넘는 포인트는 기본 완주 계산에서 제외한다.
- Path 통과율 80% 이상이며 모든 필수 Anchor를 지나야 랭킹 대상이 된다.
- 코스 랭킹은 유효한 완주 시간을 오름차순으로 정렬한다.
- 기준 시간보다 빠른 기록은 `-`와 파란색, 느린 기록은 `+`와 빨간색으로 표시한다.
- 빠른 기록에는 교통법규 준수 및 안전 운전 안내를 항상 함께 표시한다.
- 최고속도 경쟁, 속도 보상, 과속 여부 확정 표현은 제공하지 않는다.

크루장은 랭킹 방식을 다음 중 선택한다.

- `FASTEST`: 완주 시간이 짧은 순서
- `CLOSEST_TO_BASELINE`: 기준 시간과 절대 차이가 작은 순서

## 4. 일일 추천 코스

- 운영자가 준비하고 검수한 약 200개 코스를 후보 풀로 사용한다.
- 매일 모든 사용자에게 동일한 약 3개 코스를 노출한다.
- 지역과 대표 키워드가 겹치지 않도록 운영 배치가 선택한다.
- 추천 이력은 날짜와 표시 순서를 저장해 사용자마다 결과가 달라지지 않게 한다.
- 초기에는 관리자 선정 방식을 사용하고, 이용 데이터가 충분해진 뒤 자동 편성을 추가한다.

## 5. 지도 선택

- 앱 지도와 길안내 공급자는 TMAP으로 통일한다.
- 홈 지도에는 MySQL의 검수된 Cafe 좌표를 마커로 표시한다.
- 카카오 Local API는 관리자용 후보 검색·주소 보조 수단으로만 선택적으로 사용한다.
- 외부 제공자 데이터는 약관과 보존 정책을 확인하고 서비스 원본 데이터와 분리한다.

## 6. 구현 단계

### 1차 기반

- 소셜 로그인, 사용자 계정
- 카페 조회와 코스 Path
- TMAP 외부 실행용 Anchor
- 백그라운드 GPS 업로드와 완주 검증
- 5개 하단 탭 구조

### 2차 핵심 화면

- 홈 지도·카페 필터·즐겨찾기·상세
- 일일 코스 추천·코스 상세·랭킹
- 프로필·차량·주행 기록

### 3차 소셜 기능

- 커뮤니티 게시물·이미지·댓글·좋아요·신고
- 공개/초대 크루·멤버 권한·대화
- 크루 코스·랭킹 모드

## 7. 운영 전 필수 항목

- 백그라운드 위치 수집 목적·보존 기간·삭제 방법을 약관에 명시한다.
- 사진은 오브젝트 스토리지에 저장하고 DB에는 URL과 메타데이터만 둔다.
- 커뮤니티 신고·차단·삭제·운영자 제재 기능 없이 공개 출시하지 않는다.
- 개발용 ngrok 대신 HTTPS 도메인, 운영 DB, 오브젝트 스토리지, 로그·모니터링을 구성한다.
